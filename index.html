Algorithm Comparison

Explanation of Chiba, Bron-Kerbosch (with Degeneracy), and Tomita Algorithms

These three algorithms are used for finding maximal cliques in an undirected graph. A clique is a subset of vertices where every vertex is connected to every other vertex. A maximal clique is a clique that cannot be extended by adding more vertices.

1️⃣ Chiba and Nishizeki Algorithm (Chiba 1985)

This algorithm is specifically designed for enumerating cliques efficiently using edge-based processing rather than vertex-based processing.

  • Uses adjacency lists instead of adjacency matrices.
  • Orders the graph by degeneracy, processing vertices in increasing order of degree.
  • Iterates over edges and extends them to form cliques.
  • Precomputes triangle lists (3-cliques) and extends them to k-cliques.
  • Works well on sparse graphs where the maximum clique size is small.

2️⃣ Bron-Kerbosch Algorithm (with Degeneracy Ordering)

A recursive backtracking algorithm used to find all maximal cliques in an undirected graph.

  • Sorts vertices in increasing order of degree (degeneracy ordering).
  • Uses pivoting to reduce the number of recursive calls.
  • Processes high-degree vertices last, reducing the number of recursive calls.

3️⃣ Tomita’s Algorithm (Tomita 2006)

Another Bron-Kerbosch-style algorithm that improves pruning using vertex ordering heuristics.

  • Sorts vertices in descending order of degree.
  • Uses the highest-degree vertex as a pivot.
  • Branches only on vertices that are not neighbors of the pivot.
